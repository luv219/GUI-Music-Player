# PyTune Box Screenshot Guide

## Purpose
Screenshots are essential for showcasing the application in the GitHub README and portfolio presentations. They give users and recruiters an immediate sense of the app's design and functionality without needing to install it.

## Required Screenshots
1. **Light theme home screen**
   - File: `assets/screenshots/01-light-theme.png`
2. **Dark theme home screen**

3. **Playlist loaded**
   - File: `assets/screenshots/03-playlist-loaded.png`
4. **Now playing with visualizer**
   - File: `assets/screenshots/04-now-playing.png`
5. **Packaged app output**
   - File: `assets/screenshots/05-packaged-app.png`

## Screenshot Tips
- Use clean music files with safe and standard filenames.
- Avoid showing private folder paths in the playlist or title bar.
- Crop out unnecessary desktop areas; focus on the application window.
- Use a consistent window size for all screenshots.
- Avoid showing any personal information.

## How to Add to README
You can embed these screenshots in markdown like this:

```markdown
![PyTune Box Light Theme](assets/screenshots/01-light-theme.png)
```
